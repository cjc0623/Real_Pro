# 프로젝트 개요

화물운송 관리 웹 서비스.
백엔드: Spring Boot / 프론트엔드: React / 앱: Flutter WebView

---

# 핵심 규칙 (반드시 준수)

- 백엔드(Spring Boot) 코드는 절대 수정하지 않는다
- API 연동 로직(fetch, axios 호출부)은 건드리지 않는다
- UI/UX 관련 작업만 진행한다 — 스타일, 레이아웃, 컴포넌트 구조

---

# 작업 전 행동 지침

- 작업 범위가 불명확하거나 영향 범위가 넓을 경우, 반드시 먼저 질문하고 확인을 받은 후 진행한다
- 새 라이브러리나 패키지 설치가 필요하면 먼저 묻는다
- 기존 컴포넌트 구조를 변경할 경우 영향 받는 파일 목록을 먼저 알려준다
- 확신이 없는 경우 "이렇게 해도 될까요?" 하고 물어본다

---

# 스타일링 원칙

- 기존 코드에서 사용 중인 스타일링 방식을 파악하고 그 방식을 그대로 따른다
- 새로운 스타일 패턴이나 라이브러리를 임의로 도입하지 않는다
- 이미 정의된 클래스, 변수, 토큰이 있으면 새로 만들지 말고 재사용한다
- 작업 전 해당 컴포넌트의 스타일 파일을 먼저 확인한다

# Flutter WebView 고려사항

- 터치 타겟 최소 44px 이상 유지
- hover 효과 대신 active/focus 상태 사용
- 스크롤은 네이티브 스크롤 사용 (overflow: auto)
- 폰트 크기 최소 14px 이상
- position: fixed 사용 시 WebView 렌더링 이슈 주의
